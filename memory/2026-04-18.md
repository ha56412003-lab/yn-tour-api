# 2026-04-18 工作日志

## 项目：yn-tour-miniprogram 分销系统

### Bug修复

**问题2: 一键分享海报生成失效**
- 文件: `src/pages/share/share.vue`
- 原因: 直接用 `uni.request` 调 API，没带 auth token
- 修复: 改用 `get()` 工具函数，自动附加 `X-User-Token`
- 状态: ✅ 已修复

**问题3: 管理后台数据丢失**
- 原因1: `server/index.js` mount 顺序错误——`/api/admin` 在 `/api/admin/announcement` 之前注册，导致 announcement 路由被 adminAuth 中间件拦截
- 修复1: 将 `app.use('/api/admin/announcement', announcementRouter)` 移到 `app.use('/api/admin', adminRouter)` 之前
- 原因2: admin panel 前端所有 API 调用没带 `x-admin-token` header
- 修复2: 添加 `authFetch` wrapper 函数，给所有管理后台 API 调用自动附加 token
- 状态: ✅ 已修复

**问题1: 导入订单成为分销商**
- 需求: 用户可提交其他平台的订单证明，无需购买799元旅游卡即可申请成为分销商
- 流程: 用户填表单（手机号、订单号、来源平台）→ 出现在管理后台 → 管家审核 → 通过后用户自动成为分销商
- 文件: 创建 `src/pages/join/join.vue`
  - 方式一：购买799旅游卡（跳转 product 页）
  - 方式二：填写其他平台订单号提交审核
  - 查看审核进度弹窗
- 后端 API 已完整实现 (`/import/submit`, `/import/list`, `/import/approve`, `/import/reject`)
- 状态: ✅ 已完成

## 🔴 严重安全漏洞审计（进行中）

### 漏洞1: 新用户注册自动成为分销商
- **文件**: `server/routes/user.js` `/user/wx-login` 和 `/auth/phone-login`
- **问题**: 新用户创建时 `isDistributor: true`，所有新用户无需审核直接变成分销商
- **状态**: 🔴 已修复 - 改为 `isDistributor: false`

### 漏洞2: 前端登录后token丢失
- **文件**: `src/api/user.ts` 的 `wxLogin` 和 `phoneLogin`
- **问题**: API 返回 `{ user, token }`，但函数只返回 `user`，导致 `userState.token` 为空
- **影响**: 所有需要鉴权的接口调用失败
- **状态**: 🔴 已修复 - 返回值改为 `{ user, token }`

### 漏洞3: 提现接口无鉴权（未修复⚠️）
- **文件**: `server/routes/withdraw.js`
- **问题**: `/withdraw/apply` 等用户端接口没有 `userAuth` 中间件，任何人可以伪造任意用户的提现申请
- **风险**: 高 - 攻击者可指定任意userId申请提现
- **状态**: ⚠️ 待修复

### 漏洞4: 登录接口枚举用户（低危）
- **文件**: `server/routes/user.js` `/user/wx-login`, `/auth/phone-login`
- **问题**: 通过返回不同错误信息可判断手机号是否已注册
- **状态**: 低危，暂不处理

### 漏洞5: 用户信息接口未鉴权（低危）
- **文件**: `server/routes/user.js` `/user/info`
- **问题**: 可通过openid/phone查询任意用户基本信息
- **状态**: 低危，暂不处理（小程序API不对外暴露）

### 漏洞6: MCP服务器安全（之前已记录）
- OpenClaw CVE-2026-33579 已加入MEMORY.md，2026.3.28+已修复
- 公司网关需确认网络隔离
